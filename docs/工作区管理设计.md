## 一、概述

本文档描述未知叙事（Unknown Diegesis）编辑器的工作区管理方案，包括文件存储格式、目录结构、数据模型等核心设计决策。

## 二、核心决策

### 2.1 存储格式

| 决策项            | 选择                               | 原因                       |
| ----------------- | ---------------------------------- | -------------------------- |
| 数据格式          | **JSON**                           | 简单、可读、易于解析和恢复 |
| 不使用 SQLite     | 过度设计，小说数据量不大           |
| 不使用纯 Markdown | 无法表达多列布局、骰子块等复杂结构 |

### 2.2 文件粒度

| 决策项           | 选择               | 原因                         |
| ---------------- | ------------------ | ---------------------------- |
| 文件粒度         | **一章一个文件**   | 安全性高、损坏隔离、灵活导出 |
| 不使用整书单文件 | 风险大，损坏则全丢 |
| 不使用段落级文件 | 过于细碎，管理复杂 |

### 2.3 章节文件命名

| 决策项       | 选择                     | 原因                           |
| ------------ | ------------------------ | ------------------------------ |
| 文件命名方案 | **12位UUID**             | 标题独立、允许重名、引用稳定   |
| 标题存储位置 | 文件内部 meta.title      | 用户可自由修改标题，不影响文件 |
| 向后兼容     | 支持旧格式 `001-标题.ud` | 确保现有项目正常工作           |

**UUID方案优势**：

- 标题完全独立：用户输入什么就是什么，不会在文件名中强制添加编号前缀
- 允许章节重名：多个章节可以使用相同标题，文件名永不冲突
- 引用永久稳定：双链系统基于文件内部ID，不受标题修改影响
- 版本管理友好：重排序只修改内部编号，不重命名文件

### 2.4 文件后缀

| 后缀    | 全称             | 用途                     |
| ------- | ---------------- | ------------------------ |
| `.ud`   | Unknown Diegesis | 章节内容、角色卡、设定等 |
| `.json` | 标准 JSON        | 项目元数据               |

### 2.4 版本控制

**Git 管理不是刚需**。对于个人写作者，简单的自动保存 + 云同步更实用。

## 三、目录结构

```
我的小说/                    # 用户打开这个文件夹作为工作区
│
├── project.json            # 项目元数据
│
├── chapters/               # 章节文件夹
│   ├── a3f2e1d4b7c9.ud    # 第一章（12位UUID文件名）
│   ├── b4e3f2d1c8a7.ud    # 第二章
│   ├── c5f4e3d2b9a8.ud    # 第三章
│   └── ...
│
├── workspace/              # 写作资料
│   ├── characters/         # 角色卡
│   │   ├── 主角.ud
│   │   └── 反派.ud
│   ├── locations/          # 地点设定
│   │   └── 古堡.ud
│   ├── items/              # 物品设定
│   │   └── 神剑.ud
│   ├── timeline.ud         # 时间线
│   └── notes/              # 杂项笔记
│       └── 灵感.ud
│
└── assets/                 # 静态资源
    ├── images/             # 图片
    │   ├── cover.png       # 封面
    │   ├── map.png         # 地图
    │   └── characters/     # 角色图片
    │       └── protagonist.jpg
    └── documents/          # 参考文档
        └── reference.pdf
```

### 3.1 目录说明

| 目录/文件      | 说明                         |
| -------------- | ---------------------------- |
| `project.json` | 项目级元数据，每个项目唯一   |
| `chapters/`    | 存放所有章节，按编号排序     |
| `workspace/`   | 写作辅助资料，可自定义子目录 |
| `assets/`      | 二进制资源文件，不入数据库   |

## 四、文件格式规范

### 4.1 项目元数据（`project.json`）

```json
{
  "version": "1.0",
  "title": "我的小说",
  "author": "作者名",
  "description": "这是一部关于...的小说",
  "genre": "奇幻",
  "created": "2026-03-06T10:00:00Z",
  "modified": "2026-03-06T15:30:00Z",
  "statistics": {
    "wordCount": 50000,
    "chapterCount": 10,
    "characterCount": 5
  },
  "settings": {
    "autoSave": true,
    "autoSaveInterval": 3000,
    "defaultBlockType": "paragraph"
  }
}
```

#### 字段说明

| 字段          | 类型   | 必填 | 说明                     |
| ------------- | ------ | ---- | ------------------------ |
| `version`     | string | 是   | 格式版本号               |
| `title`       | string | 是   | 书名                     |
| `author`      | string | 否   | 作者名                   |
| `description` | string | 否   | 简介                     |
| `genre`       | string | 否   | 类型/流派                |
| `created`     | string | 是   | 创建时间（ISO 8601）     |
| `modified`    | string | 是   | 最后修改时间（ISO 8601） |
| `statistics`  | object | 否   | 统计信息                 |
| `settings`    | object | 否   | 项目设置                 |

---

```json
    "modified": "2026-03-06T15:30:00Z",
    "wordCount": 2500

  "blocks": [
    {
      "id": "block_1709700000000_abc123",
      "type": "heading",
      "content": "# 第一章：开端",
      "references": [],
      "referencedBy": [],
      "metadata": {
        "created": "2026-03-06T10:00:00Z",
        "modified": "2026-03-06T15:30:00Z"

    },

      "id": "block_1709700000001_def456",

        "created": "2026-03-06T10:00:00Z",

          "id": "col_2",

```

#### 元数据字段说明

| 字段        | 类型   | 必填 | 说明                             |
| ----------- | ------ | ---- | -------------------------------- |
| `id`        | string | 是   | 章节唯一标识（12位UUID）         |
| `title`     | string | 是   | 章节标题                         |
| `number`    | number | 是   | 章节序号                         |
| `status`    | string | 否   | 状态：`draft`/`revising`/`final` |
| `created`   | string | 是   | 创建时间                         |
| `modified`  | string | 是   | 最后修改时间                     |
| `wordCount` | number | 否   | 字数统计                         |

---

### 4.3 块类型定义

| 类型     | 标识符           | 说明                        |
| -------- | ---------------- | --------------------------- |
| 标题     | `heading`        | Markdown 标题（# ~ ######） |
| 段落     | `paragraph`      | 普通文本段落                |
| 引用     | `quote`          | 引用块                      |
| 无序列表 | `bulletList`     | - 开头的列表                |
| 有序列表 | `orderedList`    | 1. 开头的列表               |
| 任务列表 | `taskList`       | - [ ] 任务项                |
| 分割线   | `horizontalRule` | --- 或 \*\*\*               |
| 骰子块   | `dice`           | 掷骰功能块                  |
| 图片     | `image`          | 图片块（引用 assets/）      |

#### 骰子块特有字段

| 字段       | 类型   | 说明                         |
| ---------- | ------ | ---------------------------- |
| `formula`  | string | 骰子公式，如 `2d6+3`、`1d20` |
| `result`   | number | 掷骰结果                     |
| `rolledAt` | string | 掷骰时间                     |

### 4.4 资料文件（角色卡/地点/物品）

```json

  "type": "character",
  "meta": {
    "name": "艾莉丝",
    "role": "主角",
    "created": "2026-03-06T10:00:00Z",
    "modified": "2026-03-06T15:30:00Z"
  },
  "blocks": [
    {
      "id": "block_001",
      "type": "heading",
      "content": "# 艾莉丝"
    },
    {
      "id": "block_002",
      "type": "paragraph",

  "layoutRows": []
```

#### 资料类型

| 类型   | 标识符      | 存放目录                |
| ------ | ----------- | ----------------------- |
| 角色   | `character` | `workspace/characters/` |
| 地点   | `location`  | `workspace/locations/`  |
| 物品   | `item`      | `workspace/items/`      |
| 时间线 | `timeline`  | `workspace/`            |
| 笔记   | `note`      | `workspace/notes/`      |

## 五、双向链接系统

### 5.1 链接格式

在块内容中使用 `[[文件名#块ID]]` 语法：

```markdown
艾莉丝走进了 [[古堡.ud#block_005]]，那里是 [[反派.ud#block_001]] 的居所。
```

### 5.2 链接存储

```json
{
  "id": "block_002",
  "content": "艾莉丝走进了 [[古堡.ud#block_005]]",
  "references": [
    {
      "targetFile": "workspace/locations/古堡.ud",
      "targetBlock": "block_005",
      "type": "location"
    }
  ],
  "referencedBy": []
}
```

### 5.3 跨文件索引

编辑器维护全局索引（内存或缓存文件），支持：

- 快速查找所有引用
- 反向链接（谁引用了我）
- 断链检测

## 六、用户体验

### 6.1 核心操作

| 操作         | 行为                                                    |
| ------------ | ------------------------------------------------------- |
| **打开项目** | 选择文件夹 → 检测 `project.json` → 加载项目             |
| **新建项目** | 选择空文件夹 → 创建 `project.json` 和目录结构           |
| **新建章节** | 生成12位UUID → 创建 `chapters/{uuid}.ud` → 标题独立存储 |
| **删除章节** | 确认后删除文件 → 更新统计                               |
| **重排序**   | 拖拽调整顺序 → 仅更新内部 number，不重命名文件          |
| **新建资料** | 选择类型 → 创建对应目录下的 `.ud` 文件                  |

### 6.2 自动保存

- **防抖保存**：内容变化后 3 秒自动保存
- **定期保存**：每 5 分钟强制保存
- **失败重试**：最多 3 次，间隔 30 秒

### 6.3 导出功能（后续实现）

| 格式     | 说明                            |
| -------- | ------------------------------- |
| Markdown | 导出为 `.md` 文件，保留基本格式 |
| EPUB     | 电子书格式，支持章节划分        |
| PDF      | 打印友好格式                    |
| DOCX     | Word 文档格式                   |

## 八、设计原则

1. **简单优先**：不过度设计，满足当前需求即可
2. **用户透明**：文件格式对用户可见，出问题可手动修复
3. **渐进增强**：先实现核心功能，后续逐步扩展
4. **安全可靠**：一章一文件，损坏隔离
