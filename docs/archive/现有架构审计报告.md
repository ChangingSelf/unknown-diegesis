## 4. 风险点识别

### 4.1 高风险

1. **Schema 定义复杂**
   - LayoutRow 嵌套 LayoutColumn 嵌套 BlockWrapper
   - `content` 约束必须精确
   - **缓解**：参考 ProseMirror 文档，编写 Schema 单元测试

2. **拖拽逻辑重写**
   - 原生 drag events → @dnd-kit/core
   - Alt+拖拽创建并列
   - 边界判断复杂
   - **缓解**：分步实现，独立测试

3. **数据迁移无损**
   - 正向转换 + 反向转换
   - 边界情况处理
   - **缓解**：完整的单元测试，备份机制

### 4.2 中风险

1. **NodeView 性能**
   - 过多重渲染
   - **缓解**：使用 React.memo，优化依赖

2. **列宽调整**
   - 实时更新 Node 属性
   - **缓解**：使用 Tiptap 命令，防抖处理

3. **多标签页架构**
   - 共享 Editor 实例
   - 切换时保存/恢复
   - **缓解**：EditorContext + 状态快照

### 4.3 低风险

1. **特殊块 UI**
   - DiceBlock、ImageBlock 组件可复用
   - **缓解**：包装为 NodeView

2. **图片系统集成**
   - 服务独立，耦合度低
   - **缓解**：在 NodeView 中注入依赖
